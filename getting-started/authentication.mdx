# Authentication

The NeetoCRM API uses API keys to authenticate requests. You can view and manage your API keys in the NeetoCRM admin panel.

## Authentication Header

Include your API key in the `X-Api-Key` header of your requests:

```bash
X-Api-Key: your_api_key_here
```

## Example Request

```bash
curl -X GET "https://your-workspace.neetocrm.com/api/external/v1/contacts" \
  -H "X-Api-Key: your_api_key_here" \
  -H "Content-Type: application/json"
```

## Generating an API Key

1. Log in to your NeetoCRM workspace
2. Go to Settings > API Keys
3. Click "Create API Key"
4. Give your key a descriptive label
5. Copy the generated key (it will only be shown once)

## Security Best Practices

- Keep your API keys secret - never expose them in client-side code
- Use environment variables to store API keys
- Rotate your keys regularly
- Create separate keys for different applications

## Error Responses

| Status Code      | Description                            |
| ---------------- | -------------------------------------- |
| 401 Unauthorized | Missing or invalid API key             |
| 403 Forbidden    | Valid key but insufficient permissions |
