## API overview

The NeetoCRM API is organized around [REST](https://en.wikipedia.org/wiki/Representational_State_Transfer) principles and uses standard HTTP response codes. All API endpoints return JSON responses and require authentication via API keys.

Before you start with NeetoCRM APIs you will need:

1. [A NeetoCRM workspace](/getting-started/identifying-subdomain).
2. [An API key](/getting-started/authentication).
